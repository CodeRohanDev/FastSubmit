## ðŸ“ Styling Details

### Logo
- **Height:** 48px (h-12) on public form, 40px (h-10) in preview
- **Width:** Auto (maintains aspect ratio)
- **Object Fit:** Contain
- **Recommended Size:** 200x50px

### Company Name
- **Font Size:** 18px (text-lg) on public form, 14px (text-sm) in preview
- **Font Weight:** Semibold (font-semibold)
- **Color:** Gray-900 (light theme), White (dark theme)

### Tagline
- **Font Size:** 14px (text-sm) on public form, 12px (text-xs) in preview
- **Color:** Gray-600 (light theme), Gray-400 (dark theme)

### Container
- **Padding:** 32px top, 24px bottom (pt-8 pb-6)
- **Border:** Bottom border separating from form content
- **Background:** Matches form theme

## ðŸŽ¯ Best Practices

1. **Logo Dimensions:**
   - Use horizontal logos (wider than tall)
   - Recommended: 200x50px or similar ratio
   - Max height will be 48px, width scales proportionally

2. **Company Name:**
   - Keep it short and recognizable
   - 2-4 words ideal
   - Example: "Acme Inc.", "TechCorp", "Design Studio"

3. **Tagline:**
   - Brief and descriptive
   - 3-8 words ideal
   - Example: "Building the future", "Your trusted partner"

4. **Logo URL:**
   - Use HTTPS URLs
   - Ensure image is publicly accessible
   - Use CDN for better performance
   - Supported formats: PNG, JPG, SVG, WebP

## ðŸ”„ Responsive Behavior

### Desktop (>768px)
- Logo and text side-by-side
- Full padding and spacing
- Optimal readability

### Mobile (<768px)
- Logo and text stack vertically if needed
- Reduced padding for better space usage
- Logo scales down proportionally

## ðŸŒ“ Theme Support

### Light Theme
- Company Name: Dark gray (#111827)
- Tagline: Medium gray (#4B5563)
- Border: Light gray (#F3F4F6)

### Dark Theme
- Company Name: White (#FFFFFF)
- Tagline: Light gray (#9CA3AF)
- Border: Dark gray (#374151)

## âœ¨ Special Features

1. **Error Handling:** If logo fails to load, it's hidden automatically
2. **Conditional Rendering:** Branding section only shows if at least one field has a value
3. **Embedded Mode:** Works in both embedded and standalone forms
4. **Preview:** Real-time preview in settings page
5. **Validation:** No validation required - all fields optional
